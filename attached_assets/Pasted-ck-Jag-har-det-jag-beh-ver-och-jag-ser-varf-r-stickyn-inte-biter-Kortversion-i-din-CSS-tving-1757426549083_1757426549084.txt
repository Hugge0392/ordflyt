Kortversion: i din CSS tvingar du vänster läsyta att bli en EGEN scroll-container (overflow: auto på .reading-content.accessibility-enhanced). Då scrollar användaren inne i vänsterkolumnen – inte sidan – och din högra sticky-panel följer inte med. Det är därför den “inte hänger med”. 

index (1)

Fix (2 små ändringar)
1) Gör overflow: visible i Normal mode

I index.css har du:

.reading-content.accessibility-enhanced {
  /* ... */
  overflow: auto;   /* ← gör vänsterkolumnen till egen scroll */
  /* ... */
}


Byt till:

/* Normal mode (default): INGEN egen scroll i läsytan */
.reading-content.accessibility-enhanced {
  overflow: visible;
}

/* Om du vill behålla intern scroll i Fokuserat läge, scopea så här: */
.focus-mode .reading-content.accessibility-enhanced {
  overflow: auto;
}


(Om du inte har en .focus-mode-klass på wrappern i fokusläget ännu, kan du lägga till den där du renderar FocusMode.) 

index (1)

2) Behåll/stärk sticky-wrappen runt frågorna

Din NormalMode gör faktiskt rätt redan – du har sticky top-8 på själva frågekontainern i högerspalten. Fortsätt så här (valfritt: lägg till self-start på kolumnen så den inte försöker sträckas):

<div className="reading-main-grid grid grid-cols-1 lg:grid-cols-[3fr_1fr] gap-8 items-start">
  {/* Vänsterkolumnen: texten */}
  {/* ... */}

  {/* Högerkolumnen: sticky panel */}
  <div className="reading-questions-column order-2 lg:order-2 self-start">
    <div
      className="questions-panel-container sticky top-8 z-50 border rounded-lg p-6"
      /* ... */
    >
      {/* ... */}
    </div>
  </div>
</div>


Detta matchar redan din komponent (du har sticky top-8), så den kritiska delen är att inte låsa scrollen i vänsterkolumnen. 

NormalMode (1)

Varför detta fungerar

position: sticky fäster mot den scrollande föräldern. Just nu scrollar användaren i vänster .reading-content.accessibility-enhanced (eftersom CSS tvingar overflow: auto), medan frågepanelen ligger utanför den scroll-containern. När du släpper den regeln i normal-läget låter du sidan (viewporten) vara scrollcontainern och då klistrar sticky top-8 som förväntat.

Bonus: långt innehåll i frågepanelen

Om frågorna kan bli högre än skärmen, ge panelens inre scroll, inte hela sidan:

<div className="questions-panel-container sticky top-8">
  <div className="max-h-[calc(100vh-2rem)] overflow-auto">
    {/* panelens innehåll */}
  </div>
</div>


Det gör att panelen stannar men dess innehåll kan scrollas.

Kör de två ändringarna ovan så ska panelen sitta fast som du tänkt. Om den fortfarande inte fastnar efter det, kolla snabbt att ingen förälder till panelen har overflow-y: hidden/auto/scroll eller någon backdrop-filter/transform på sig (de kan sabotera sticky). Men i din kodbas är det den där overflow: auto på läsytan som är boven.
